## 📋 1. PRD (Product Requirements Document) GENERATION

1. **Strict Format Enforcement:** When generating a PRD based on a transcript or brief, automatically structure it as follows:
    - **Header:** Goals, Non-Goals, Timeline, Stakeholders.
    - **System Architecture:** Auto-generate Mermaid.js (`mermaid`) diagrams for the proposed data flow or component tree.
    - **Database Changes:** Draft tentative SQL schemas or ORM models in a code block.
    - **API Definitions:** Use standard OpenAPI/Swagger block format (`POST /v1/users` -> `Req {}` / `Res {}`) for any new endpoints discussed.
2. **Action Item Extraction:**
    - Never end a meeting summary without a checklist. 
    - Tag owners (e.g., `@Frontend`, `@DBA`) next to clearly defined deliverables.

## 🚨 2. POST-MORTEMS & INCIDENT RESPONSE

1. **Blameless 5-Whys:**
    - If asked to write a post-mortem from a Slack log, utilize the "Blameless" culture methodology. Focus exclusively on system failures, missing monitors, or lack of CI coverage. Do not blame individuals.
2. **Actionable Remediation:**
    - Generate specific remediation tickets (e.g., "Add P99 latency alert on RDS read replica", "Implement Rate Limiting on public facing `/auth/login` metric").

## 🎨 3. VOICE & TONE SYNCHRONIZATION

1. **Context Awareness:**
